## вҡЎ Quick Start

> **Requirements:** Node.js 18+, npm 9+

```bash
# 1. Clone the repository
git clone https://github.com/TSCREATES-copyright/colorharmony.git

# 2. Enter the project directory
cd colorharmony

# 3. Install dependencies
npm install

# 4. Start the development server
npm run dev

# 5. Open in browser
# вҶ’ http://localhost:5173
```

That's it. No `.env` file. No API keys. No backend setup. No account creation.

Open the browser, paste a hex color, pick a harmony rule, and your first palette is ready in under 10 seconds.

<br />

### Build for Production

```bash
# Build optimized static output
npm run build

# Preview the production build locally
npm run preview
```

The output is a fully static site. Deploy anywhere: GitHub Pages, Netlify, Vercel, Cloudflare Pages вҖ” or just open `dist/index.html` directly in a browser.

```bash
# Deploy to GitHub Pages (optional)
npm run deploy

# Deploy to Netlify (optional)
netlify deploy --prod --dir=dist
```

**Zero runtime AI. Zero external API calls. Zero accounts. Zero latency from network.**

## рҹ“җ Data Model

```typescript
// в”Җв”Җв”Җ Harmony Rules в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

export type HarmonyRule =
  | 'analogous'       // hue offset Вұ20вҖ“40В°, soft and cohesive
  | 'complementary'   // hue offset 180В°, high contrast, brand-strong
  | 'triad'           // hue offset Вұ120В°, three-way balance
  | 'tetrad'          // hue offset Вұ90В°/180В°, four-point harmony
  | 'monochrome';     // one hue, varied lightness and saturation

// в”Җв”Җв”Җ Swatch Role в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

export type SwatchRole =
  | 'base'            // Primary brand color вҖ” anchor of the palette
  | 'accent'          // Interactive and highlight elements
  | 'support'         // Hover, active, secondary states
  | 'neutral'         // Mid-ground вҖ” backgrounds, dividers
  | 'background'      // Page and surface backgrounds
  | 'text';           // Body and heading text

// в”Җв”Җв”Җ Swatch в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

export type PaletteSwatch = {
  id: string;
  role: SwatchRole;
  hex: string;
  rgb: [number, number, number];
  hsl: [number, number, number];
  contrastOnWhite: number;       // WCAG ratio against #FFFFFF
  contrastOnBlack: number;       // WCAG ratio against #000000
  name: string;                  // Human-readable color name
  wcagAA: boolean;               // Passes WCAG 2.1 AA (4.5:1)
  wcagAAA: boolean;              // Passes WCAG 2.1 AAA (7:1)
  locked?: boolean;              // Locked swatches survive regeneration
};

// в”Җв”Җв”Җ Palette Record в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

export type PaletteRecord = {
  id: string;
  title: string;
  baseColor: string;
  rule: HarmonyRule;
  swatches: PaletteSwatch[];
  score: number;                 // 0вҖ“100 composite harmony score
  tags: string[];
  notes?: string;
  createdAt: number;             // Unix timestamp
  updatedAt: number;
  version: number;               // Incremented on each save
};

// в”Җв”Җв”Җ App Settings в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

export type AppSettings = {
  theme: 'light' | 'dark';
  defaultRule: HarmonyRule;
  defaultExportFormat: 'css' | 'json';
  dailyUsageCount: number;
  dailyUsageResetDate: string;
  premiumUnlocked: boolean;
  lastOpenedPaletteId?: string;
};
```

## рҹҺЁ Harmony Rule Reference

A complete guide to when and why to use each harmony mode:

<br />

### Analogous вҖ” `Вұ20В° to Вұ40В° hue offset`

```
COLOR WHEEL POSITION:

        60В°
       в–Ҳв–Ҳ
   в–Ҳв–Ҳ      в–Ҳв–Ҳ
  40В°      80В°
  (three adjacent hues)

Example: #3B82F6 (Blue)
Output:  #3B82F6  вҶ’  #3B68F6  вҶ’  #5A3BF6  вҶ’  #6B5BF6  вҶ’  #1E40AF
```

**Best for:** Backgrounds, UI surfaces, content-heavy layouts, editorial design, calm brand identities, healthcare, fintech.

**Produces:** Cohesive, low-tension palettes. Colors feel related and peaceful. Safe for large surface areas.

**Avoid for:** High-energy brands, attention-demanding CTAs, anything needing strong contrast between elements.

<br />

### Complementary вҖ” `180В° hue offset`

```
COLOR WHEEL POSITION:

  в–Ҳв–Ҳ в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ в–Ҳв–Ҳ
  0В°                  180В°
  (directly opposite)

Example: #3B82F6 (Blue)
Output:  #3B82F6 (Blue)  +  #F6923B (Orange) as complementary pair
         Supporting tones derived from both anchor hues
```

**Best for:** Brand identities, CTAs, hero sections, product packaging, anything needing strong visual tension.

**Produces:** High-contrast, energetic palettes. The complementary pair creates natural hierarchy вҖ” one color always reads as action, one as context.

**Avoid for:** Large text-heavy surfaces where sustained visual tension causes fatigue.

<br />

### Triad вҖ” `Вұ120В° hue offset`

```
COLOR WHEEL POSITION:

       в–Ҳв–Ҳ
      /  \
     /    \
   в–Ҳв–Ҳ      в–Ҳв–Ҳ
  (equidistant triangle)

Example: #3B82F6 (Blue)
Output:  #3B82F6 (Blue)  вҶ’  #F63B82 (Pink)  вҶ’  #82F63B (Green)
```

**Best for:** Creative brands, gaming, entertainment, children's products, social media graphics, anything needing vibrant multi-color energy.

**Produces:** Balanced but vibrant palettes. Three equally weighted hues create visual richness without any single color dominating.

**Avoid for:** Conservative or minimal brands, clinical interfaces, heavy data displays.

<br />

### Tetrad вҖ” `Вұ90В° and 180В° hue offsets`

```
COLOR WHEEL POSITION:

  в–Ҳв–Ҳв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв–Ҳв–Ҳ
  |            |
  |            |
  в–Ҳв–Ҳв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв–Ҳв–Ҳ
  (rectangular/square)

Example: #3B82F6 (Blue)
Output:  #3B82F6 (Blue)  вҶ’  #82F63B (Green)  вҶ’  #F63B82 (Pink)  вҶ’  #F6823B (Orange)
```

**Best for:** Complex design systems, multi-section landing pages, dashboards with multiple state types, component libraries needing semantic color diversity.

**Produces:** Rich, multi-faceted palettes. Most complex to balance вҖ” the engine handles saturation and lightness adjustments to keep all four hues from competing equally.

**Avoid for:** Minimal brands, single-product contexts, small-scale interfaces.

<br />

### Monochrome вҖ” `Single hue, varied L/S`

```
COLOR WHEEL POSITION:

  в–Ҳв–Ҳ (one hue)

Lightness/Saturation axis:
  в–‘в–‘в–‘в–‘в–‘в–’в–’в–’в–’в–“в–“в–“в–“в–Ҳв–Ҳв–Ҳв–Ҳ (dark to light)

Example: #3B82F6 (Blue)
Output:  #1E3A5F  вҶ’  #2B5EA7  вҶ’  #3B82F6  вҶ’  #93C5FD  вҶ’  #EFF6FF
```

**Best for:** Sophisticated brands, premium products, minimalist design systems, dark mode interfaces, photography portfolios, finance and legal products.

**Produces:** The most elegant and restrained palette type. Highly coherent. Every swatch feels deliberate. Works beautifully when combined with a strong neutral system.

**Avoid for:** Multi-state UIs needing semantic color distinction, anything requiring clear separation between different data categories.

## рҹӣЎпёҸ Validation System

The ValidationSystem runs after every generation to catch issues before they reach the UI:

```
CHECK 1: Input Validation
  вң“  Is HEX exactly 3 or 6 characters?
  вң“  Are RGB values clamped to 0вҖ“255?
  вң“  Is HSL hue within 0вҖ“360В°?
  вң“  Is saturation and lightness within 0вҖ“100%?

CHECK 2: Palette Integrity
  вң“  Are all 5 role slots assigned?
  вң“  Are there duplicate hex values?
  вң“  Is there sufficient lightness separation between swatches?
  вң“  Does the palette include at least one dark and one light swatch?

CHECK 3: Contrast Safety
  вң“  Does text swatch pass 4.5:1 contrast on background swatch?
  вң“  Does accent swatch pass 3:1 contrast on base swatch?
  вң“  Are any critical pairs flagged below AA?

CHECK 4: Rule Consistency
  вң“  Do generated hue offsets match the selected rule within tolerance?
  вң“  Is the base swatch preserved as the anchor?
  вң“  Are locked swatches unchanged from the previous generation?
```

Validation failures produce actionable warnings вҖ” not silent corrections. The user always knows what changed and why.
